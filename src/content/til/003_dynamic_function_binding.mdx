`ld.so`, the dynamic linker on unix systems, is a shared object
itself, which bekons the question of how `ld.so` was every dynamically
loaded itself, if shared objects require a dynamic linker in order to
be loaded into a program at runtime.

`ld.so` uses _self relocating code_.
